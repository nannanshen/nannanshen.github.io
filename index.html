HTTP请求源IP伪造

  • 有些目标站点配置了只允许特定IP访问，但目标验证源IP不够严苛的情况下，可以通过伪造IP来达到访问某些web应用的目的。利用前提是目标站点判断IP是通过请求头获取的。

出现点

  • 网站管理后台

伪造方法

  • 在HTTP请求包中加入特定请求头和值

X-Forwarded-For: 192.168.0.91
X-Originating-IP: 192.168.0.92
X-Remote-IP: 192.168.0.93
X-Remote-Addr: 192.168.0.94
X-Client-IP: 192.168.0.95

可利用工具

  • burp
  • 插件

拓展

  • 不知道确切IP的时候，可以通过轮询的方法。已知网站是某个地区的，可以先找出该地区的IP段，用IP段中的IP一个个尝试。

windows提权之UAC绕过

发表于 2020-02-27

UAC

  • 用户账户控制，是Windows Vista及更高版本操作系统中采用的一种控制机制。
  • 一旦程序设计系统更改或是特定任务，除非以管理员权限运行，否则这些操作都将被阻止，没有管理员权限将不能进行以下操作：
    • 注册表修改
    • 加载设备驱动程序
    • DLL注入
    • 修改系统时间
    • 修改用户账户控制
    • 修改受保护的目录
    • 计划任务
  • 参考文章：https://www.hackingarticles.in/multiple-ways-to-bypass-uac-using-metasploit/

利用以下模块：search bypassuac

  3. set session id
  4. exploit以后会创建一个新的session
  5. session id+1

七个模块

模块都是用use命令使用，用info查看模块详细信息。show options查看配置信息。search搜索模块。

  1. exploit

    漏洞利用模块。对漏洞进行利用。

    • 使用方法：use 模块名，show target 查看模块支持的目标环境
    • 先对目标进行扫描，能够利用再执行攻击模块
    • 反弹回的是cmdshell可以更改payload返回一个meterpreter

  2. payloads

    攻击载荷模块。可以执行反弹代码，添加用户等攻击。

    1. 正向payload：会被防火墙拦截

      • 攻击者去连接目标服务器

      • 使用场景：

        • 目标及其不给其他除特定端口外的端口出站
        • 目标处于深层网络，不能直接连通外网，其他机器开了防火墙，也不能通过其他机器连通，通过开着防火墙的机器端口转发达到穿透目的

        关闭防火墙：netsh firewall set opmode mode=disable

        • msfvenom -p linux/x86/meterpreter/bind_tcp LPORT=<Attack Port> -f elf > /root/bind_xx.elf

webshell管理工具

发表于 2020-02-21 更新于 2020-02-22

中国菜刀（China chopper）

文件管理

  • 上传webshell的时候注意编码格式，一般选用UTF-8。连接的时候也可选择，匹配就行。打开中文文件乱码就改成GB2312再连接。
  • 用webshell进行文件管理的时候出错一般是权限问题。权限继承于解析器（web容器，web服务器），这些解析器的权限是管理员配置的。Apache默认是system|administrator权限，Nginx，IIS，Tengine默认是Network Service（NT Service）。权限低的时候，比如freehost虚拟主机，除了网站本身的目录其他都不能访问。
  • 访问目录以后中国菜刀会自动缓存网页的目录结构，即使以后webshell掉线也依然能查看到。网站变更以后再用webshell访问需要清空缓存。
  • 如果上传文件失败或者文件过大，可以通过下载的方式从远程服务器下载文件到网站目录。文件上传或下载要注意编码格式。
  • 下载文件建议不下载过大的文件，如果要下载，可以将文件复制到网站根目录，然后用浏览器访问该文件下载。这样比用菜刀下载要快很多。大文件的复制最好使用命令行copy过去，用菜刀的虚拟终端执行命令。

数据库管理

  • 进去以后要点一下配置，可以选择数据库连接类型以及一些连接配置，一般数据库的配置在网站目录下的config文件里能找到。提交正确的配置以后，菜刀会根据你的权限列出所有能操作的表。
  • 通过双击表名或者字段名菜刀会写好查询语句，点一下执行就可以查看数据。也可以自己修改sql语句执行。双击列出的数据可以对内容进行修改、复制等操作。右键可以导出数据，切记不能乱脱库。

00截断绕过

  • 条件：php版本小于5.3.4–CVE-2006-7243，phpmagic_quotes_gpc为OFF状态
  • GET和POST请求进行截断时的区别：post不会像get对%00进行解码，所以需要在十六进制中手动修改。
  • 路径可控可在路径截断，文件名可控可在文件名截断。1.php%00

解析漏洞绕过

  • 双重扩展名：apache 文件名解析时，是从后面开始检查后缀，按最后一个合法后缀执行。如： shell.php.xxx，xxx不会被解析，这个文件会被当做php文件解析。黑名单比较好用，白名单条件比较苛刻。
  • iis6.0：*.asp;1.jpg
  • iis7.0/7.5/Nginx<8.03：php.ini⾥默认cgi.fix_pathinfo=1，对其进行访问的时候，在URL路径后添加.php后缀名会当做php文件进行解析。上传1.jpg，访问http://www.xxx.com/1.jpg/.php，此时1.jpg会被当做PHP脚本文件来解析。

      访问1.jpg/.php ，那么1.jpg就会被解析执行。

Apach

  • 不可识别解析后缀

    • Apache的文件解析过程是从右到左开始判断解析,如果为不可识别的后缀解析,就再往左判断。
    • 在Apache的解析中，除了“php|php3|phtml”等规定的后缀中，任何的后缀加入，都是不会被识别解析的，也会被跳过后缀处理。test.php.k1.w2.a3 -> test.php

  • .htaccess

    • htaccess⽂件是Apache服务器中的一个配置文件，它负责相关目录下的网页配置。通过htaccess文件，可以帮我们实现：网页301重定向、自定义404错误页面、改变文件扩展名、允许/阻止特定的用户或者目录的访问、禁止目录列表、配置默认文档等功能。

    • 该漏洞适用于目标web服务器为apache，并且.htaccess可以被上传执行的情况。

    • 当.htaccess⽂件内容为SetHandler application/x-httpd-php时，即设置当前目录所有文件都使⽤PHP解析，那么无论上传任何文件，只要文件内容符合PHP语言代码规范，就会被当作PHP执行。不符合则报错。

    • 在Apache中如果需要启动 .htaccess，必须在http.conf中设置 AllowOverride

      • <FilesMatch "1.jpg"> SetHandler application/x-httpd-php </FilesMatch>

双文件上传和竞争上传

双文件上传

  • 应用场景
    • 南方/良精CMS
    • 正则默认匹配第一个filename的情况
  • 利用原理
    • 服务端默认取第二个文件作为真正的上传文件，但是在检测时只检测了第一个文件。或者只验证了第一个filename的拓展名。
  • 利用方法
    • 直接在审查元素中添加一个上传表单
    • burp抓包之后在数据包中添加

竞争上传

  • 应用场景：访问webshell的时候，短时间内可以访问到，再次访问就访问不到了。
  • 利用原理：代码逻辑为，先将文件上传到服务器，然后检测后缀名，如果不符合再通过unlink删除文件，因此可以通过条件竞争的方式在unlink之前，访问webshell。
  • 利用方法：使用burp不停上传，同时用python脚本不停访问上传的文件。上传的文件访问到以后会生成一个一句话文件，只要访问成功一次就能写入。

sql注入工具sqlmap的使用

发表于 2020-02-20

注入Access

  • 一般小型网站使用asp+Access
  • Access的特性，没办法直接获取表名列名，只能通过暴力猜解。一般使用sqlmap自带的就行。
  • 一个小技巧，在网站后台页面查看登录框源码可能会有字段名出现。

一般流程

  • -u 检测注入，–dbs 爆数据库，–curren-db 获取当前数据库，–tables 爆表名，–columns 爆列名，–dump爆数据，–start 1 –stop 2 指定第一条到第二条数据，测试一两条数据，挖src的时候不容易出事。
  • 拿到密码解密后可以登录后台，无后台可以用这个密码尝试登录远程桌面或者ftp服务

注入mysql

  • 基础用法同上

sqlmap参数

  • -r参数，从一个文件里获取一个HTTP请求，这样就不需要对cookie等请求头参数进行设置。sqlmap会自己对这些参数进行测试 。sqlmap -r 文件物理路径
  • -m参数，从文件里一条一条获取url，一个一个进行检测。也就是批量检测
  • –cookie参数，有时候登录以后的注入要带上cookie才能检测。
  • –radom-agent参数，随机更改user-agent，防止被识别出来。
  • –proxy参数，使用代理，不使用真实ip，可以翻墙，也可以隐藏身份。
  • –delay参数，设置两次请求之间的延时。防止请求速度过快被封。–delay 1
  • –tamper参数，指定绕过脚本，对注入语句进行变形以达到绕过waf的目的。
  • –technique参数，指定注入技术，B布尔，E报错，U联合，S堆叠，T时间。
  • –risk参数，提高测试等级，最高为3。会增加测试语句。
  • –lever参数，增加被测试的语句。
  • –string参数，在正常的页面找一个错误页面没有的字符串，添加到参数后面帮助识别注入。注：在填写搜索关键词的时候，一定要找出那些能够搜索出内容的关键词，比如你在实战中搜索dark5，但是网站没有关于dark5的文章内容，就不会出现正常的搜索结果，这样我们把注入语句加在dark5的后面时，是没有办法判断我们的SQL语句是否成功执行，就没有办法达到我们想要的结果了。还有的网站使用ajax，有时候真正的请求会在后面，需要通过抓包来判断
  • *，这个参数用来指定测试语句位置，通常用于伪静态页面，也可用于正常页面增加效率。文章：https://www.cnblogs.com/ximotao/p/5870274.html
